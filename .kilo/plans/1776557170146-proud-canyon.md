# Project Structure Plan - Open Creatrweb

## Overview
Scaffold a fully static HTML/CSS/JS field guide to free open-source AI tools with personalized creative direction, built exclusively with free models to demonstrate accessible web creation.

## Design Goals
- **Accessibility**: Semantic HTML5, ARIA labels, keyboard navigation, WCAG AA contrast
- **Open Canvas Layout**: Asymmetric, expressive, surprising use of space across all viewports
- **Personalization**: Owner's voice and visual style embedded throughout
- **Static Architecture**: No backend, no database, no server-side logic
- **Progressive Enhancement**: JavaScript adds interactivity but pages remain functional without it

## Key Implementation Decisions

### 1. Semantic HTML5 Structure
- All pages use proper semantic elements (header, nav, main, section, article, footer)
- ARIA landmarks for accessibility
- Keyboard navigation support built into foundation

### 2. Open Canvas Layout Foundation
- Flexible grid systems supporting asymmetric layouts
- Creative spacing and positioning
- Equally expressive across mobile, tablet, and desktop

### 3. Modular CSS Architecture
- main.css for site-specific styles and open canvas layout
- Vendor directory for CSS framework (to be selected via gallery)
- Separate concerns: layout, typography, components, utilities

### 4. Progressive Enhancement JavaScript
- Modular files for different functionality areas
- Scroll-triggered interactions for open canvas experience
- Tool filtering capabilities
- UI component interactions

### 5. Content Organization
- Clear separation between tools, guides, and informational pages
- Tool card-based homepage for exploration
- Individual tool pages with detailed information
- Curated guides for learning pathways

## Personalization Strategy

### Visual Identity
- **Color Palette**: Derived from fornesusart.com abstract work (gallery to select)
- **Typography**: Characterful fonts with personality (gallery to select)
- **Layout**: Open canvas approach reflecting abstract expressionist principles
- **Assets**: Abstract art from fornesusart.com as visual anchors

### Voice and Tone
- **Writing Style**: First-person, present tense, unguarded and reflective (as per fornesus.blog)
- **Copy Governance**: AI-generated drafts for owner editing, not replacement
- **Editorial Voice**: Direct without polish, intimate but never indulgent

### Interactive Elements
- **Scroll-triggered reveals**: Purposeful interactions carrying meaning
- **Tool filtering**: Interactive exploration of tool ecosystem
- **Dynamic layouts**: Open canvas responsiveness across viewports

## Accessibility Implementation

### Semantic Structure
- Proper heading hierarchy (h1-h6)
- Semantic HTML5 elements for content meaning
- ARIA roles and landmarks

### Keyboard Navigation
- Tab order logical and intuitive
- Skip links for main content
- Focus management for interactive elements

### Visual Accessibility
- WCAG AA contrast ratios
- Focus indicators visible and clear
- Resizable text without loss of functionality

### Screen Reader Support
- Alt text for all meaningful images
- Descriptive link text
- Proper form labels and instructions

## Technical Constraints

### Static Architecture
- No server-side includes or build processes
- Pure HTML/CSS/JS for maximum portability
- No package managers or dependencies

### Free Model Stack Only
- All code generated by confirmed free models only
- No paid or closed-source model output
- Document any vendor dependencies in docs/dependencies.md

### URL Permanence
- Finalize file and URL structure before creating pages
- No breaking changes once published
- Permanent redirects for any moved content
